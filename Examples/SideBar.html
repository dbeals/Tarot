Tarot is a lightweight, mobile-first, card-based boilerplate framework written using LESS. We've split our design into 3 different principals: cards, modifiers, plugins.

Cards

The core elements in Tarot are designed around card: content, navigation bars, etc. The other elements like lists and forms, are designed around being placed in a card. This helps keep our elements focused and easily managed while giving a sleek design.

Modifiers

Almost all of the styling is broken down into different components that can be applied separately to different elements. This way you can change the background of any element to the primary color just by adding the class "primary-back" or change the border color using "primary-border". While this does lead to some bloat, the framework itself is still very lightweight (about 20kb as of writing this.)

Plugins

The framework itself contains the basic components to allow you to handle simple prototyping. There is no navigation or JavaScript usage in the core - all of this will be included via plugins. So, while you can't just tag something "nav-bar" to get all the styling applied, you can easily make something using the modifiers available, the same way that this example page does. For something more advanced and full-featured, we'll be adding a side-bar plugin.
